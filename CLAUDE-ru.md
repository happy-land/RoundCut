## Обзор проекта

RoundCut - это российская система управления резкой и ценообразованием стали с React фронтендом и NestJS бэкендом. Приложение управляет ценообразованием стали, складскими запасами, операциями резки и расчетами наценок для стальных изделий.

## Архитектура

### Фронтенд (React + TypeScript)
- **Инструмент сборки**: Vite с React + TypeScript
- **Управление состоянием**: Redux Toolkit с RTK Query для API вызовов
- **Стилизация**: SCSS с некоторыми Tailwind CSS, пользовательские шрифты (OpenSans, IBM Plex Mono)
- **UI компоненты**: Material-UI, Headless UI, MDB React UI Kit
- **Ключевые функции**: Фильтрация марок стали, выбор диаметра, расчет цен, управление складом

### Бэкенд (NestJS + TypeScript)
- **Фреймворк**: NestJS с TypeScript
- **База данных**: PostgreSQL с TypeORM
- **Аутентификация**: JWT с Passport.js (локальная и JWT стратегии)
- **Ключевые модули**: Пользователи, Позиции прайса, Склады, Наценки, Резки, Категории, CSV импорт

## Команды разработки

### Фронтенд (из директории /frontend)
```bash

npm run build        # Сборка для продакшена (tsc + vite build)
npm run lint         # Запуск ESLint
npm run preview      # Предварительный просмотр продакшен сборки
```

### Бэкенд (из директории /backend)
```bash

npm run start        # Запуск продакшен сервера

npm run test:e2e     # Запуск end-to-end тестов
```

## Ключевые концепции предметной области

### Основные сущности
- **PriceItem**: Стальное изделие с ценовыми уровнями (за 1т, 5т, 15т), размерами, весом
- **Warehouse**: Места хранения с связанными наценками
- **Cut**: Операции резки стальных изделий
- **Markup**: Уровни ценовой наценки (уровень1-уровень8) на склад
- **Category**: Система категоризации продукции

### Поток данных
1. Импорт CSV данных о ценах через эндпоинт `/csv`
2. Фильтрация позиций прайса по марке стали и диаметру
3. Применение расчетов наценок на основе склада и количества
4. Генерация позиций резки для конкретных длин и весов

## Паттерны структуры файлов

### Фронтенд
- `src/components/` - React компоненты, организованные по функциям
- `src/features/` - Redux слайсы и управление состоянием
- `src/services/` - Определения RTK Query API
- `src/utils/` - Утилитарные функции и определения типов
- `src/pages/` - Компоненты маршрутов

### Бэкенд
- `src/[module]/` - NestJS модули с контроллерами, сервисами, сущностями, DTO
- `src/auth/` - Гарды аутентификации и стратегии
- `src/utils/` - Общие утилиты и типы

## Конфигурация базы данных

Использует PostgreSQL с переменными окружения:
- `POSTGRES_HOST`, `POSTGRES_PORT`, `POSTGRES_USER`, `POSTGRES_PASSWORD`
- `POSTGRES_DB`, `POSTGRES_SCHEMA`

## Аутентификация

- JWT-аутентификация с Passport.js
- Локальная стратегия для логина username/password
- JWT стратегия для защищенных маршрутов
- Идентификация пользователей по email

## Тестирование

- Бэкенд использует Jest для юнит и e2e тестов
- Фронтенд использует встроенные возможности тестирования Vite
- Тестовые файлы следуют конвенции `*.spec.ts`

## Важные замечания

- Приложение в основном на русском языке
- Стальные изделия используют метрические единицы измерения (мм, кг, тонны)
- Расчеты цен поддерживают скидки на основе количества
- Выбор склада влияет на расчеты наценок
- Функциональность CSV импорта для массового обновления цен
